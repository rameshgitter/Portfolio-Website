üéØ Step-by-Step Contribution Process

1

Read Contributing Guidelines

Click on the "contributing guidelines" link shown in the banner on the issues page. This document contains important information about:

  • Code style and standards
  • Testing requirements
  • Commit message format
  • PR (Pull Request) submission process
üí° Tip: Take notes while reading. This will save you time later!
2

Choose an Issue

Look for issues labeled as:

  • good first issue - Perfect for beginners
  • Easy - Simple to implement
  • dx - Developer experience improvements

From the screenshot, good options include:

  • #25 - Progress Bar for Training
  • #24 - Model Summary API
3

Comment on the Issue

Before starting work, comment on the issue:

Hi! I'm interested in working on this issue. This would be my first contribution to the project. Could you confirm if it's still available and if there are any specific requirements I should know about?
‚ö†Ô∏è Important: Always check if someone is already working on the issue. Wait for maintainer confirmation before starting.
4

Fork the Repository

Click the "Fork" button at the top right of the repository page. This creates your own copy of the project.

5

Clone Your Fork

Open your terminal and run:

git clone https://github.com/YOUR-USERNAME/etna.git cd etna

Replace YOUR-USERNAME with your GitHub username.

6

Set Up Remote Upstream

Connect your fork to the original repository:

git remote add upstream https://github.com/etsi-ai/etna.git git remote -v

This allows you to sync with the original project later.

7

Create a New Branch

Never work on the main branch. Create a feature branch:

git checkout -b feature/issue-number-short-description

Example: git checkout -b feature/25-progress-bar

8

Set Up Development Environment

Install dependencies (usually in requirements or setup files):

python -m venv venv source venv/bin/activate # On Windows: venv\Scripts\activate pip install -e .[dev] # or: pip install -r requirements.txt
9

Make Your Changes

Now write your code! Remember to:

  • Follow the project's code style
  • Write clear, descriptive variable names
  • Add comments for complex logic
  • Keep changes focused on the issue
üí° Tip: Commit frequently with clear messages. It's easier to track your progress!
10

Write Tests

Most projects require tests for new features:

  • Look for existing test files in the tests/ directory
  • Follow the same pattern for your tests
  • Run tests locally: pytest or python -m pytest
11

Commit Your Changes

Stage and commit your changes:

git add . git commit -m "feat: add progress bar for training (fixes #25)"

Common commit prefixes:

  • feat: - New feature
  • fix: - Bug fix
  • docs: - Documentation
  • test: - Tests
  • refactor: - Code refactoring
12

Sync with Upstream

Before pushing, make sure you're up to date:

git fetch upstream git rebase upstream/main

Resolve any conflicts if they appear.

13

Push to Your Fork

git push origin feature/25-progress-bar
14

Create a Pull Request

Go to your fork on GitHub. You'll see a prompt to create a PR. Click "Compare & pull request".

In your PR description, include:

  • What changes you made
  • Why you made them
  • Reference the issue: "Fixes #25"
  • Screenshots if UI changes
  • Testing steps you performed
15

Respond to Feedback

Maintainers may request changes:

  • Be patient and respectful
  • Ask questions if unclear
  • Make requested changes in your branch
  • Push updates (they'll appear in the PR automatically)
üí° Tip: Code reviews are learning opportunities. Don't take feedback personally!

‚ö†Ô∏è Common Pitfalls to Avoid

Don't start coding without claiming the issue

Someone else might be working on it, wasting your time.

Don't work on the main branch

Always create a feature branch for your changes.

Don't make massive changes in one PR

Keep PRs focused on one issue. Smaller PRs are easier to review.

Don't ignore CI/CD failures

If automated tests fail, fix them before requesting review.

Don't be discouraged by rejection

Not all PRs get merged. Learn from feedback and try again!

üéâ After Your PR is Merged

Congratulations! You're now an open source contributor!

Clean up your local repository:

git checkout main git pull upstream main git branch -d feature/25-progress-bar

Next steps:

  • Look for more issues to work on
  • Help review other people's PRs
  • Join the project's community chat if available
  • Share your contribution on social media!
